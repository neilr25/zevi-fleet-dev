# ZEVI Fleet Prototype — Dev Environment

Interactive fleet-first dashboard for Smart Green Shipping's FastRig wind-assisted propulsion service. This is the **public dev environment** for testing the API-backed, object-model-driven version of the prototype.

## What it is

A single-page prototype showing:
- Global fleet view with status, alerts, and filters
- Ship-level drill-down with FastFix, FastReport, FastRegs, and FastRoute tabs
- Object-model-grounded synthetic data (Silver / Bronze / Gold tiers)
- Real AtoBviaC illustrative routes, embedded as static data
- Ships positioned on their actual routes
- Light / dark mode toggle

## Data model

`data/fleet.json` is generated by `scripts/generate-fleet.js` and follows the ZEVI object model:

- **Silver**: vessels, deployments, FastRig units, components, sensors, contracts, voyages, routes
- **Bronze**: events (sensor readings, diagnostics, control events, downtime)
- **Gold**: performance products, PAYS invoices

The generator is seeded (`seed: 42`) so the dataset is reproducible.

## Files

- `index.html` - the prototype (loads fleet data from `/api/fleet` with static fallback)
- `index.static.html` - pure static fallback (no API dependency)
- `Logo_only.png` - SGS logo
- `data/fleet.json` - generated synthetic fleet database
- `api/fleet.js` - Vercel function returning the fleet list with full ship details
- `api/route.js` - Vercel function returning a route for a voyage
- `api/vessel/[id].js` - Vercel function returning the full object graph for one vessel
- `api/deployment/[id].js` - Vercel function returning deployment-scoped data
- `api/alerts.js` - Vercel function returning Bronze diagnostic alerts
- `scripts/generate-fleet.js` - deterministic synthetic data generator
- `scripts/test-api.js` - local API smoke test
- `scripts/test-server.js` - local server that serves the HTML and API together
- `ROADMAP.md` - proposed evolution plan
- `USER_GUIDE.md` - how to use the dashboard

## Run locally

```bash
node scripts/test-server.js
```

Then open http://localhost:3000.

To regenerate the synthetic data:

```bash
node scripts/generate-fleet.js
```

To test the API directly:

```bash
node scripts/test-api.js
```

To run the static fallback only:

```bash
python -m http.server 8088
```

## Notes

- The AtoBviaC API key is no longer embedded in the frontend HTML. Routes are stored as static AtoBviaC illustrative data in `data/fleet.json`.
- The dashboard loads the vessel list and ship details from `/api/fleet` and falls back to the embedded static dataset if the API is unavailable.
- `index.static.html` is a pure static fallback that can be renamed to `index.html` for instant rollback.
- For production, replace the JSON file with a real database (Supabase/Postgres is the natural choice) and add a backend proxy to fetch fresh AtoBviaC routes on demand.
- Route data is cached in the browser's localStorage for 7 days (legacy behaviour kept for compatibility).
